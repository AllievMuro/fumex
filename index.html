Automatic dependency resolution

Pass a transaction signature and Fumex walks the entire CPI call chain, identifying every account touched at every invocation depth — program accounts, PDAs, oracle feeds, and sysvars.

All 47 accounts are fetched in a single getMultipleAccounts batch. Zero manual --clone flags required.

Deterministic clock & slot injection

Time-sensitive programs like Marginfi collateral checks and Raydium CLMM tick boundary logic require exact slot and timestamp values. Fumex pins these at fork time.

Every test run — locally or in CI — executes against the exact same temporal context. No flaky oracle staleness errors, ever.

Features

Everything the Solana test harness was missing

CPI Trace Streaming

Decoded instruction names, account roles, and lamport deltas per invocation depth streamed to stdout or your CI runner in real time.

Pyth & Switchboard Oracles

Price feed accounts cloned at fork time, pinned deterministically. No stale price errors, no custom mock feeds to maintain.

Replayable Fixtures

Commit deterministic JSON snapshots to your repo. Any teammate reproduces byte-for-byte account state without touching a live RPC endpoint.

BPF Program Introspection

BPF Loader v2/v3 program accounts resolved and cloned automatically — no need to separately specify executable accounts in your test setup.

Protocol Resolvers (Q3)

Automatic account inclusion for Jupiter, Kamino, Drift, Marginfi, and Raydium — tick arrays, bank vaults, and auxiliary accounts identified without manual specification.

GitHub Actions Native

Published npm package with documented CI integration. Hosted snapshot cache in Q3 eliminates live RPC calls from your pipeline entirely.
